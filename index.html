龙虾焦虑

Posted on 2026-03-10 In Thinking

这里的龙虾，不是吃的龙虾，而是OpenClaw，现在AI圈已经把“龙虾（OpenClaw）”从一个技术段子，炒成了全民跟风、排队安装、甚至地方出补贴的荒诞热潮，和N年前的币圈一样，但本质是：真正需要AI的人本来就会用龙虾，需要求人装龙虾的人，其实根本用不上。

  1. AI理论价值极大，但大众实际利用率极低，很多人只是把AI当聊天工具。
  2. 按Token计费的AI工具，只适合高时间成本的人（如程序员），怂恿普通人用，本质是厂商和云平台的收割。
  3. 现在的AI炒作，FOMO（怕错过）本身成了需求，用上龙虾比用龙虾干什么更重要，和当年Web3、币圈套路如出一辙。
  4. 连周鸿祎都开始做“一键装龙虾”，说明这波炒作基本快到顶了。你就不怕红衣教主把你的隐私扒的底裤都不剩？

其实2026年AI已经进入三大分化：

  • 人群分化：前沿研究者极速迭代，普通人依旧无感，认知完全脱节。
  • 国内厂商分化：大厂烧钱冲ChatBot的DAU，小厂放弃应用，扎堆出海卖API赚钱。
  • 中美分化：地缘导致全球峰会“假装中国不存在”，但技术和用户又深度互相依赖。

所以啊：
可以好奇去试龙虾，但别为了跟风焦虑。AI是用来提效的，不是反过来把人绑架、让人更累、被工具支配的。

是你用AI，不是AI用你。

新换的房子还不错，如果早上6点清洁阿姨不来打扫就更好了。

对于一个花费了近五十万装修自己家，其他时间都是租房的人来说，目前总结了几个小的tips（出租房版本）：

  1. 免钉胶非常好用，除了乳胶漆的墙面，其他地方都可以搞定；
  2. 不管多大的衣柜或者储物格，还是要备一定数量的储物盒子，宜家的非常好用；
  3. 路由器、光猫、无线的设计装修的时候就要考虑好，不然…大部分出租屋无线都是随意摆放的，很碍眼，信号也不好；
  4. 好的床垫非常重要，即使是出租屋，也一定要配一个比较好的床垫；
  5. 一定要注意卫生间的地漏，还有厨房的台盆，这是屋子里面臭味的主要来源；
  6. 窗户要有窗纱，没有的话就买一个紫外线的杀虫仪；
  7. 要注意空调的空洞，是否堵住，不然冬天冷死，夏天热死，还有各种虫子；

租到合适的房子真的是随缘了，每次到一个新的城市，总是要花费一定的试错成本在租房子上。

教育运营型公司

Posted on 2026-03-03 In Thinking

经历了这么多互联网公司，技术团队可能遇到的两种管理风格，尤其是在一些传统的教育互联网公司里面可能更加常见：：
一种是强管控、防风险、抠工时、防个例；
一种是重信任、给空间、看结果、激发创造力。

  • 偏教育式管理（强管控）：
    把技术人员当“老师/员工”管，追求不出错、可监控、流程严密、防风险优先。
  • 偏互联网式管理（松管控）：
    把技术人员当“人才”管，追求效率、产出、创造力、信任优先。

6个关键差别

1. 工时与考勤

偏教育（强管控）

  • 严格上下班、打卡、加班记录、工时统计
  • 关注你在不在、做没做、花了多久
  • 制度为了管住少数不自觉的人，让所有人一起买单
  • 典型代表：传统线下起家的教育机构里的技术部门（早期新东方、好传统校区思维的团队）

偏互联网（信任型）

  • 弹性上下班、不盯考勤、看结果不看时长
  • 关注你交付了什么、解决了什么问题
  • 相信大多数人自觉，不为少数人牺牲整体效率
  • 典型代表：作业帮、猿辅导、好未来后期技术中台

2. 流程与审批

偏教育

  • 流程极细、审批极长
  • 制度设计逻辑：先假设你会出错/违规
  • 宁可慢，不能乱
  • 技术改个东西要多层确认，怕担责

偏互联网

  • 流程轻量化、授权充分
  • 假设你是专业的、能负责
  • 允许小范围试错
  • 技术有较大决策权，快速迭代

3. 风险偏好

偏教育

风险厌恶型

  • 宁可不做，不要做错
  • 重视合规、稳定、安全、不出舆情
  • 技术创新会被严格约束

偏互联网

风险可控下的创新型

  • 允许试错，鼓励探索
  • 技术驱动业务，而不是被动支持
  • 追求效率、体验、壁垒

4. 对技术人员的定位

偏教育

技术是支持部门、成本中心

  • 听话、稳定、不出事最重要
  • 创造力不重要，执行力最重要

偏互联网

技术是核心生产力、增长引擎

  • 技术决定产品上限
  • 鼓励主动思考、优化、创造价值

5. 沟通与决策方式

偏教育

自上而下，领导定规则
技术人员执行 > 思考
会议多、汇报多、强调规范

偏互联网

扁平沟通，数据驱动
技术人员参与决策
讲逻辑、讲效率、讲结果

6. 文化氛围

偏教育

严谨、规范、讲秩序
像学校/机关

偏互联网

开放、自由、讲结果
像真正的科技公司

为什么教育行业容易走向“强管控、防个例”？

教育基因公司的底层思维

教育行业本质是：

  • 高风险（家长投诉、舆情）

  • 高责任（未成年人）

  • 高合规

  • 规模化靠“复制”

因此教育组织天然强调：

  • 稳定

  • 可控

  • 标准化

  • 防风险

  • 规章制度覆盖极端情况

它的管理哲学是：

“不能出错”比“跑得快”更重要。

所以就会出现：

  • 工时精确管理

  • 流程层层审批

  • 每个动作都要留痕

  • 规章制度是为“极端个例”设计的

这是教育组织的“安全导向型管理”。

互联网基因公司的底层思维

互联网行业本质是：

  • 竞争激烈

  • 产品驱动

  • 不确定性高

  • 创新优先

它的组织逻辑是：

  • 试错

  • 快速迭代

  • 鼓励冒险

  • 结果导向

管理哲学是：

“持续创新”比“零风险”更重要。

所以：

  • 不精细管工时

  • 不为了1%异常限制99%人

  • 规则是为了“效率”而不是“防错”

  • 更强调目标与结果

这是“增长导向型管理”。

所以很多纯教育背景（偏重于上课的老师/校长类型）的管理者管技术时，会不自觉套用：
盯工时 + 强流程 + 防风险
而不是：
给空间 + 看结果 + 激发创造力

猪系思维

Posted on 2026-03-02 In Thinking

成年人最高级的活法：学会“猪系思维”

你发现没有？

越聪明的人，越容易内耗。
越敏感的人，越容易焦虑。
越上进的人，越容易失眠。

反而有一种生物，活得异常稳定——

猪。

它不焦虑、不自责、不反思人生、不纠结关系。
吃得下，睡得着，长得壮。

很多人把猪当成“愚钝”的象征。
但如果你仔细观察，你会发现——

猪活成了低内耗的典范。

而成年人真正的成熟，不是变复杂，
而是学会“猪系思维”。

一、猪从不反复复盘情绪

人类有一种能力，叫“精神反刍”。

一句话可以回味三天。
一个表情可以脑补十集。
一次失误可以羞愧半年。

但猪不会。

猪不会因为主人语气重了一点，就怀疑自己是不是不够优秀。
猪不会躺在猪圈里想“刚刚是不是该多表现一下”。

事情过去了，就过去了。

情绪反刍，是成年人最隐形的自我消耗。

真正强大的人，不是没有情绪，
而是不让情绪循环播放。

二、猪只对“可控的事”负责

猪的世界很简单：

  • 饿了就吃

  • 困了就睡

  • 该长肉就长肉

它不操心天气。
不操心明天。
不操心别人对它的评价。

而人类的问题是：

  • 为三年后的事焦虑

  • 为别人的态度揣测

  • 为无法掌控的变量失眠

内耗的本质，是对不可控事务的过度负责。

猪系思维的核心是：

只对自己的行为负责，不对外界结果过度负责。

在职场中尤其重要：

  • 你能控制的是努力程度

  • 不是领导的偏好

  • 更不是所有人的看法

当你把注意力拉回“可控区”，焦虑会立刻下降。

三、猪不和别的猪比

你见过猪在猪圈里内卷吗？

“它比我白一点。”
“它吃得比我多。”
“它好像更受欢迎。”

没有。

猪只负责把自己活好。

而成年人活在横向对比里：

  • 同龄人年薪多少？

  • 同事晋升多快？

  • 别人买房买车了吗？

比较，是焦虑最稳定的来源。

横向比较制造焦虑，纵向成长制造力量。

真正的成长，是今天比昨天好一点点。
不是比谁快，而是比自己稳。

四、猪对外界评价不敏感

猪不在乎别人怎么看它。

你说它胖，它继续吃。
你说它懒，它继续睡。

它没有“自我攻击系统”。

成年人最大的问题之一是——

我们太容易把“别人说的话”
变成“自我定义”。

别人一句无心之言，
你却内心风暴三天。

猪系思维提醒我们：

别人的评价，只代表他们的认知层级。

你的人生，不需要被所有人理解。

五、猪的能量管理，比人类高级

猪不会熬夜内卷。
不会带着焦虑吃饭。
不会情绪压抑到身体失衡。

而人类常常：

  • 睡眠不足

  • 情绪过载

  • 精神紧绷

你以为自己在努力。
其实是在透支神经系统。

低内耗的核心不是摆烂，
而是——

把能量留给重要的事情。

真正高效的人，不是做得多，
而是消耗少。

六、如何在生活中培养“猪系思维”？

给你5个具体练习：

  1. 停止二次复盘情绪（事情过了就过了）

  2. 每天只抓3件最重要的事

  3. 不为未发生的事提前焦虑

  4. 减少无意义社交信息摄入

  5. 保证睡眠优先级高于KPI

记住一句话：

情绪稳定，是一个人最高级的竞争力。

（以下是我认为应该做到的，实际我一个都没做到。）

很多人以为，成熟是变得更聪明。
其实成熟是变得更稳定。

学一点猪的笃定，
少一点人类的内耗。

当你不再反复攻击自己，
不再过度解释，
不再向所有人证明——

你会突然发现：

生活没有那么难。

低内耗，不是迟钝。
是节能。

而真正的强大，
是把能量留给值得的事。

AI PPT

Posted on 2026-02-09 In Work

经过我这一年的疯狂实践，总结出来一套非常有用且快速制作PPT的流程。

生成大纲

首先是一个PPT的主题，输入给各种大模型，比如ChatGPT、Gemini、豆包、Kimi等，然后将各个大模型的回答进行汇总、总结、提炼，针对其中可能的问题再进行二次提问，最终形成一个文字稿的PPT大纲。

生成初版PPT

将大纲给到豆包，直接生成ppt，这里注意，如果有想要的格式，一般要提前在prompt里面写清楚：

  1. 整体色调；
  2. 风格样式；
  3. 字体样式；
  4. …

其实一开始我是通过Gemini 的Nano Banana通过我的文字直接生成图片，因为豆瓣对接的AiPPT太难用了，大纲生成的完全不对。后来发现豆瓣改进了生成PPT的功能，可以通过对话来编辑PPT，而且速度很快，所以就直接将文字版本大纲丢给豆包了。

PPT优化

最后就是再优化一下文字、顺序、样式等，所有的都可以通过豆包的对话来完成。

当然也可以再继续增加新的内容，不过新增内容太多的话（十几轮对话之后或者增加附件内容进行解析），后续就会任务失败，原因不明。

总结

整个技术方案大概迭代了4、5次，免费付费的都用过，目前是用下来最稳定的方案，生成大纲的主力是ChatGPT，其他是辅助。

目前的方案，我在1个小时内可以写出来将近100页的高质量PPT。

后续如果有更好的方案我再更新。
